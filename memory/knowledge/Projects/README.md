# Projects

**What belongs here:** Specific, time-bound outcomes with a defined endpoint.

- Active projects with deadlines
- One-time initiatives with clear deliverables
- Campaigns or launches with a defined start/end
- Client engagements with a finite scope
- Anything that has a "done" state

**What does NOT belong here:**
- Ongoing areas of responsibility (→ Areas)
- Reference materials (→ Resources)
- Completed or abandoned projects (→ Archives)

**Memory rule:** When a project is complete or abandoned, move to Archives and extract any durable lessons to the relevant Area or Resources folder.
