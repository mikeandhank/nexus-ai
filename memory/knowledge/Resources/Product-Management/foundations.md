# Product Management — Foundations

## Core Definition

Product management is the function of building the right products for customers. It includes discovery, prioritization, delivery, and iteration.

---

## Core Responsibilities

| Area | Description |
|------|-------------|
| **Discovery** | Understand customer needs |
| **Strategy** | Define vision and roadmap |
| **Prioritization** | Decide what to build |
| **Delivery** | Ship products |
| **Iteration** | Learn and improve |

---

## Frameworks

| Framework | Description |
|-----------|-------------|
| **Jobs to be Done** | Customer jobs, not features |
| **RICE** | Prioritization scoring |
| **OKRs** | Objectives and Key Results |
| **Lean Startup** | Build-Measure-Learn |
| **Continuous Discovery** | Ongoing customer research |

---

## Key Metrics

| Metric | Definition |
|--------|------------|
| **NPS** | Net Promoter Score |
| **Activation** | % who experience value |
| **Retention** | % who stay |
| **Churn** | % who leave |
| **NPM** | Net Product Margin |

---

## Core Principles

1. **Solve real problems** — Not invented ones
2. **Data informs, doesn't decide**
3. **Launch fast, iterate**
4. **Say no often**
5. **User research is essential**
