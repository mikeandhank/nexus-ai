# Product Management — Contrarian & Case Studies

## Contrarian

### What Most People Get Wrong

- **"More features = more value"** → Less is more
- **"Users know what they want"** → They don't, observe them
- **"Build what we can"** → Build what customers need
- **"Launch date is sacred"** → Quality over dates
- **"Metrics tell the truth"** → Metrics can mislead

## 5 Successes

### 1. Apple iPhone

**What happened:** Reinvented phone.

**Core insight:**
- Simplicity wins
- Experience over specs
- Software + hardware

### 2. Slack

**What happened:** Built product people love.

**Core insight:**
- Small team, big impact
- Fast iteration
- Listen to users

### 3. Notion

**What happened:** All-in-one workspace.

**Core insight:**
- Simple, flexible
- User feedback shapes product
- Product-led growth

### 4. Figma

**What happened:** Design tool for everyone.

**Core insight:**
- Collaboration is feature
- Browser-first
- Real-time sync

### 5. Amazon

**What happened:** Customer-obsessed product.

**Core insight:**
- Working backwards
- Two-pizza teams
- Data-driven decisions

## 5 Failures

### 1. Feature Factories

**Building features nobody wants → waste**

### 2. Waterfall

**Long cycles without feedback → wrong product**

### 3. Tech Debt

**Skipping fundamentals → slows everything**

### 4. No User Research

**Guessing instead of asking → wrong product**

### 5. Scope Creep

**Too much in release → delayed, buggy
