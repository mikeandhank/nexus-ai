# Machine Learning — Foundations

## Core Definition

Machine Learning (ML) is a subset of AI that enables computers to learn patterns from data and produce models for predictions or decisions without explicit programming.

---

## Core Concepts

| Concept | Description |
|---------|-------------|
| **Algorithm** | Step-by-step rules for processing data (e.g., linear regression, decision trees) |
| **Model** | Trained output encapsulating learned patterns |
| **Training** | Minimizing loss via optimization |
| **Testing** | Evaluating on held-out data to avoid overfitting |
| **Evaluation** | Metrics like accuracy, F1 score, precision, recall |

---

## Main Types of ML

### Supervised Learning
- Uses labeled training data (input-output pairs)
- Learns mapping for prediction or classification
- Examples: Linear regression, Naïve Bayes, Decision trees, Random forests

### Unsupervised Learning
- Analyzes unlabeled data to find hidden structures
- Examples: K-means clustering, PCA (dimension reduction)

### Reinforcement Learning
- Agents learn via trial-and-error interactions with environment
- Optimizes for rewards
- Examples: Q-learning, Deep Q-Networks (DQN)

### Deep Learning
- Subset of ML using multi-layered neural networks
- Excels at complex, hierarchical data (images, speech, text)
- Requires more data and compute than traditional ML
- Examples: CNNs, RNNs, Transformers

## The ML Workflow

1. **Define Business Problem** — What are we solving?
2. **Data Collection** — Gather relevant data
3. **Data Preprocessing** — Clean, label, feature engineer
4. **Model Selection** — Choose algorithm
5. **Training** — Fit model to data
6. **Evaluation** — Test on held-out data
7. **Deployment** — Put into production
8. **Monitoring** — Track performance, detect drift

---

## Key Terms

| Term | Definition |
|------|------------|
| **Overfitting** | Model learns training data too closely, fails on new data |
| **Underfitting** | Model too simple to capture patterns |
| **Feature Engineering** | Selecting/transforming input variables |
| **Hyperparameter** | Settings before training (e.g., learning rate) |
| **Training Data** | Data used to fit the model |
| **Test Data** | Held-out data for final evaluation |
| **Bias-Variance Tradeoff** | Balance between underfitting and overfitting |
